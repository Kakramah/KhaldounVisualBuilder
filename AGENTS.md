# AGENTS.md — دستور مشروع KhaldounVisualBuilder

هذا المستند موجَّه لأي وكيل ذكي (Claude / GPT / Cursor / غيره) يعمل داخل هذا المجلد.
اقرأه **قبل** أي توليد.

---

## 1. مصدر الإدخال

- **`input.md`** هو المصدر الوحيد للنص.
- لا تجلب نصاً من أي مكان آخر (لا من الذاكرة، لا من الإنترنت، لا من ملفات أخرى).
- إذا كان `input.md` فارغاً أو يحوي القالب الافتراضي فقط، أوقف العمل واطلب من المستخدم لصق نصه.

### 1أ. التعامل مع frontmatter داخل `input.md`

كثير من نصوص خلدون تأتي مع رأس metadata:
```
**التاريخ:** ...
**رمز الأرشيف:** SRA.v###
**المصدر:** ...
**الزوج البصري:** prompts/...
```

طبّق هذه القواعد عند البناء:
- **التاريخ** و**رمز الأرشيف** و**الموقع الجغرافي** يُعرَضان في الموقع (شريط أرشيف علوي + footer).
- **المسارات الداخلية** (`prompts/...`, `posts/...`) تُسقَط — هي مراجع لنظام خلدون لا للموقع.
- إذا لا يوجد frontmatter، لا تخترع واحداً.

## 2. قواعد الأسلوب والهوية

- **`skills/`** يحوي قواعد خلدون عكرمة (snapshot من نظامه الحي في Drive):
  - `01-identity-and-values.md` — الهوية، التوقيع، القيم، الخط الأحمر.
  - `02-language-and-tone.md` — 9 قوانين Anti-AI العربي + 5 نبرات.
  - `03-visual-system.md` — Control Gates + Patch v1.3 + المراجع البصرية.
  - `04-image-prompts.md` — معيار GPT Image 2 + نظام DNA.
  - `05-web-translation.md` — HTML/CSS الأساسي.
  - `06-prohibitions.md` — 7 أخطاء موثقة + الخطوط الحمراء.
  - `07-sources.md` — روابط Drive لـ 30+ ملف مرجعي ضخم.
- اقرأ `00-index.md` ثم 01..06 بهذا الترتيب. لا تُعدِّل ملفات `skills/` أثناء البناء.
- اللغة المُولَّدة آلياً تخضع لقواعد `02-language` بحرفها (مثلاً: لا em dash). النص **الأصلي** المنقول من `input.md` يُحترَم كما كتبه خلدون.

## 3. مكان المخرجات

- **`output/`** هو المجلد الوحيد للنشر. كل ما داخله يجب أن يكون جاهزاً للرفع على **GitHub Pages** كما هو.
- الهيكل المتوقع بعد البناء:
  ```
  output/
  ├── index.html
  ├── style.css
  ├── poster.html (اختياري)
  └── images/
      ├── hero.png
      ├── scene-01.png
      ├── ...
      └── image_prompts.md   ← محلي فقط، لا يُربط من HTML
  ```

## 4. قواعد إلزامية (Hard Rules)

| # | القاعدة | السبب |
|---|---------|------|
| 1 | الصفحات العامة **لا تعرض أي برومبت صورة أبداً** | البرومبتات أدوات إنتاج داخلية |
| 2 | برومبتات الصور **فقط** في `output/images/image_prompts.md` | فصل الإنتاج عن النشر |
| 3 | كل مراجع الصور في HTML/CSS **مسارات نسبية** (`images/hero.png`) | يعمل على GitHub Pages بدون كسر |
| 4 | **ممنوع** `file:///` أو `/Users/...` أو أي مسار مطلق محلي | يكسر النشر ويسرّب مسار المستخدم |
| 5 | اللغة الافتراضية **عربي** مع `<html lang="ar" dir="rtl">` | هوية المشروع |
| 6 | النمط البصري: **Khaldoun cinematic documentary** (راجع `skills/`) | اتساق الهوية |
| 7 | لا بيانات شخصية للمستخدم (إيميل، اسم نظام، مسارات) في المخرج | خصوصية + قابلية النشر |
| 8 | التوقيع: نص `— Khaldoun A. Akramah` في footer + داخل برومبت كل صورة `@𝓚𝓱𝓪𝓵𝓭𝓸𝓾𝓷𝓐𝓪𝓀𝓻𝓪𝓶𝓪𝓱` bottom-right | راجع `01-identity` |
| 9 | الأداة المولّدة للصور: **GPT Image 2** فقط، نثر إنجليزي، لا معاملات Midjourney | راجع `04-image-prompts` و `06-prohibitions` |

## 5. ما يُولَّد في كل تشغيل

استخدم نظام **Tasks** لتتبع الخطوات (TaskCreate/TaskUpdate). الترتيب الإلزامي:

1. **لصق + قراءة `input.md`** — تأكد من وجود نص حقيقي.
2. **اختيار التصنيف البصري** (1 ثقيل / 2 رسمي / 3 خفيف / 4 شاعري) — قرار صامت، يُكتب أعلى `image_prompts.md`.
3. **`output/index.html`** — موقع عربي RTL متجاوب يعكس نص `input.md` كاملاً.
4. **`output/style.css`** — يطبّق لوحة التصنيف المختار + قواعد الـ RTL والتيبوغرافيا العربية.
5. **`output/images/image_prompts.md`** — برومبت لكل صورة (Hero + 2–5 مشاهد) مع:
   - `aspect_ratio` صريح في رأس البرومبت (مثل `3:2`, `16:9`, `4:3`).
   - DNA code كامل (`TYPE-SUBJECT-EMOTION-LIGHT-...`).
   - Caption_energy العربي قبل البرومبت.
   - التوقيع `@𝓚𝓱𝓪𝓵𝓭𝓸𝓾𝓷𝓐𝓪𝓀𝓻𝓪𝓶𝓪𝓱` في نهاية كل برومبت.
6. **Placeholders للصور** — PNG حقيقية بنفس النسب المذكورة في البرومبتات (داكنة + اسم الصورة)، حتى يعرض الموقع التخطيط الصحيح قبل التوليد. **ليست `.txt`** — يجب أن تكون PNG قابلة للعرض.
7. **اختياري:** `output/poster.html` إذا النص يستحق بوستراً مستقلاً (2:3).

## 6. ما لا تفعله

- لا تنشئ ملفات خارج `output/` و`skills/` و`input.md` و`AGENTS.md` و`README.md`.
- لا تكتب README جديد أو ملفات توثيق إضافية إلا بطلب صريح.
- لا تضع تعليقات داخل HTML تكشف عن البرومبتات أو منطق التوليد.
- لا تستخدم CDN غير ضروري أو سكربتات تتبع.
- لا تنشر تلقائياً (لا `git push`, لا `gh-pages deploy`) — انتظر إذن المستخدم.
- لا تقرأ ملفات Drive الضخمة كاملةً أثناء البناء (راجع قاعدة الاقتصاد في `01-identity`). استخدم `skills/07-sources.md` كروابط مرجعية فقط.

## 7. عند الانتهاء

اعرض للمستخدم:
- شجرة الملفات الجديدة في `output/`.
- التصنيف البصري المختار + اللوحة اللونية.
- عدد الصور المطلوب توليدها يدوياً ومسار `image_prompts.md`.
- خطوات النشر على GitHub Pages (دون تنفيذها).
- أي ثغرة أو قرار اضطررت لاتخاذه بمفردك أثناء البناء (شفافية مع المستخدم).
