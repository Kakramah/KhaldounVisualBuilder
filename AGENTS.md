# AGENTS.md — دستور مشروع KhaldounVisualBuilder

هذا المستند موجَّه لأي وكيل ذكي (Claude / GPT / Cursor / غيره) يعمل داخل هذا المجلد.
اقرأه **قبل** أي توليد.

> **النطاق الحالي:** بناء **مواقع HTML احترافية بهوية خلدون البصرية**. لا يبني توليد صور ولا برومبتات GPT Image (تلك تتم في جلسة منفصلة عند خلدون). الصور تأتي جاهزة من المستخدم أو من نظامه السابق.

---

## 1. البنية العامة

```
KhaldounVisualBuilder/
├── input.md         ← النص الخام (المصدر الوحيد للمحتوى)
├── AGENTS.md        ← هذا الملف
├── README.md        ← دليل المستخدم
├── skills/          ← قواعد خلدون (snapshot من Drive)
├── templates/       ← قوالب HTML+CSS+README جاهزة للنسخ
├── examples/        ← أعمال سابقة مكتملة (للمرجعية والإلهام)
└── output/          ← البناء الفعلي الحالي (يُنشر على GitHub Pages)
```

## 2. مصدر الإدخال

- **`input.md`** هو المصدر الوحيد للنص.
- لا تجلب نصاً من أي مكان آخر.
- إذا كان `input.md` فارغاً أو يحوي القالب الافتراضي فقط، أوقف العمل واطلب من المستخدم لصق نصه.

### 2أ. التعامل مع frontmatter داخل `input.md`

نصوص خلدون كثيراً ما تأتي مع رأس metadata:
```
**التاريخ:** ...
**رمز الأرشيف:** SRA.v###
**المصدر:** ...
**الزوج البصري:** prompts/...
```

- **التاريخ + رمز الأرشيف + الموقع الجغرافي** يُعرَضون في الموقع (شريط أرشيف أو footer).
- **المسارات الداخلية** (`prompts/...`, `posts/...`) تُسقَط — مراجع نظام خلدون لا للموقع.
- إذا لا frontmatter، لا تخترع واحداً.

## 3. مكان المخرجات

- **`output/`** هو المجلد الوحيد للنشر الحالي. كل ما داخله جاهز للرفع على **GitHub Pages**.
- الهيكل المتوقع:
  ```
  output/
  ├── index.html
  ├── style.css
  ├── poster.html (اختياري إن كان النص يستحق غلافاً مستقلاً)
  └── images/
      ├── hero.png
      ├── scene-01.png
      └── ...
  ```
- **الصور تأتي جاهزة** (يضعها المستخدم أو يولّدها في جلسة منفصلة). الوكيل **لا يولّد صوراً** ولا يكتب image_prompts.md.

## 4. قواعد الأسلوب والهوية (skills/)

`skills/` يحوي قواعد خلدون كـ snapshot من Drive:

| الملف | متى يُقرأ |
|------|-----------|
| `00-index.md` | دائماً — خريطة الترتيب |
| `01-identity-and-values.md` | دائماً — التوقيع، الألوان، الخط الأحمر |
| `02-language-and-tone.md` | عند كتابة أي نص عربي للجمهور (9 قوانين + 5 نبرات) |
| `03-visual-system.md` | عند اختيار لوحة/تكوين/مزاج بصري للويب |
| `05-web-translation.md` | HTML/CSS الأساسي + قواعد RTL |
| `06-prohibitions.md` | دائماً — 7 أخطاء موثقة + خطوط حمراء |
| `07-sources.md` | عند الحاجة لتفصيل غير موجود — روابط Drive |

اقرأ `00-index.md` ثم 01..06 بهذا الترتيب. لا تُعدِّل ملفات `skills/` أثناء البناء.

> **قاعدة em dash:** اللغة المُولَّدة آلياً تخضع لـ `02-language` بحرفها (لا em dash). النص **الأصلي** من خلدون في `input.md` يُحترَم كما كتبه.

## 5. القوالب (templates/)

`templates/` يحوي قوالب جاهزة للنسخ. كل قالب مجلد فيه:
- `index.html` — هيكل HTML مع تعليقات توجيهية في الأماكن القابلة للتعبئة.
- `style.css` — CSS متوافق مع نظام خلدون البصري.
- `README.md` — متى تستخدمه، علامات النص المناسب، كيف تستخدمه.

**القوالب المتاحة:**

| # | اسم | متى |
|---|-----|-----|
| 01 | manifesto | نص يُعلن موقفاً، ضربة افتتاحية + جسم + ضربة نهاية |

> ملاحظة: عمدنا إلى عدم بناء قوالب احتياطية مسبقاً. حين تطلب نوعاً جديداً (memorial, archive-entry, document, single-statement, event...) يُبنى وقتها بمواصفات النص الفعلي، لا بتخمين.

## 6. الأعمال السابقة (examples/)

`examples/` يحوي بنايات سابقة كاملة كمرجع وإلهام:

| المثال | النوع | متى تنظر إليه |
|--------|------|-------------|
| `hareetan/` | archive-entry (إحياء ذكرى) | عند بناء صفحة تخليد ذكرى أو منشور أرشيفي |

لا تعدّل `examples/` أبداً. لها قيمة كحالة زمنية محفوظة.

## 7. ما يُولَّد في كل تشغيل

استخدم نظام **Tasks** لتتبع الخطوات (TaskCreate/TaskUpdate). الترتيب:

1. **اقرأ `input.md`** كاملاً + frontmatter إن وُجد.
2. **اختر التصنيف البصري** (1 ثقيل / 2 رسمي / 3 خفيف / 4 شاعري) — قرار صامت، يُكتب كتعليق في style.css.
3. **اختر القالب الأنسب** من `templates/` أو أنشئ بنية جديدة إن لم يطابق نص-المستخدم أيّاً منها.
4. **انسخ القالب إلى `output/`** ثم استبدل العناوين والفقرات بنص `input.md`.
5. **عدّل style.css** ليطابق التصنيف البصري المختار (لوحة لونية + تيبوغرافيا + إيقاع).
6. **رتّب الصور المتوفرة** في `output/images/` (إن وُجدت) — استخدم classes `.wide-figure` و `.inset-figure` و `.hero-media` بحسب الموقع.
7. **اختياري:** `poster.html` إذا النص يستحق غلافاً مستقلاً.

## 8. قواعد إلزامية (Hard Rules)

| # | القاعدة | السبب |
|---|---------|------|
| 1 | كل مراجع الصور في HTML/CSS **مسارات نسبية** (`images/hero.png`) | يعمل على GitHub Pages بدون كسر |
| 2 | **ممنوع** `file:///` أو `/Users/...` أو أي مسار مطلق محلي | يكسر النشر ويسرّب مسار المستخدم |
| 3 | اللغة الافتراضية **عربي** مع `<html lang="ar" dir="rtl">` | هوية المشروع |
| 4 | النمط البصري: **Khaldoun cinematic documentary** | اتساق الهوية |
| 5 | لا بيانات شخصية للمستخدم (إيميل، اسم نظام، مسارات) في المخرج | خصوصية + قابلية النشر |
| 6 | التوقيع: نص `— Khaldoun A. Akramah` في footer كل صفحة | راجع `01-identity` |
| 7 | كل `<img>` لها `alt` عربي وصفي (للوصولية) | accessibility |
| 8 | Semantic HTML5 (`<main>`, `<article>`, `<section>`, `<figure>`) | احتراف + SEO |

## 9. ما لا تفعله

- لا تنشئ ملفات خارج البنية أعلاه.
- لا تكتب README جديد أو توثيق إضافي إلا بطلب صريح.
- لا تضع تعليقات HTML تكشف منطق التوليد.
- لا تستخدم CDN غير ضروري أو سكربتات تتبع.
- لا تستبدل ملفاً في `output/` بدون قراءته أولاً (احذر `cp` فوق ملف لم تراه).
- لا تنشر تلقائياً (`git push`, `gh-pages deploy`) — انتظر إذن المستخدم.
- لا تقرأ ملفات Drive الضخمة كاملةً أثناء البناء (راجع قاعدة الاقتصاد في `01-identity`).

## 10. عند الانتهاء

اعرض للمستخدم:
- شجرة الملفات في `output/`.
- التصنيف البصري المختار + اللوحة اللونية.
- القالب المستخدم من `templates/` (إن وُجد).
- خطوات النشر على GitHub Pages (دون تنفيذها).
- أي قرار اضطررت لاتخاذه بمفردك (شفافية).
